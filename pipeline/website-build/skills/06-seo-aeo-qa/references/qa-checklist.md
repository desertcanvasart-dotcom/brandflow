# Pre-Launch SEO/AEO + QA Checklist

Mark each: Pass / Fail / N/A, with a note. Fails need a specific fix.

## 1. Technical SEO
- [ ] Unique, intentional title tag on every page
- [ ] Unique meta description on every page
- [ ] One clear H1 per page; logical H2/H3 hierarchy
- [ ] Canonical tags correct
- [ ] sitemap.xml present and submitted
- [ ] robots.txt correct (nothing important blocked)
- [ ] 301 redirects from all old URLs (if replacing a site)
- [ ] No broken internal links
- [ ] Clean, readable URL slugs

## 2. AEO / Structured Data
- [ ] Schema markup present and valid (Organization, and page-appropriate types: FAQ, Article, Product, LocalBusiness, etc.)
- [ ] Business entity is unambiguous to machines (name, location, category consistent everywhere)
- [ ] Each AEO target question from the Dig Brief has a page/section that answers it clearly and extractably
- [ ] Answer-shaped content where relevant (definitions, steps, comparisons, FAQs)
- [ ] Content structured so AI engines can quote it (clear headings, self-contained answers)

## 3. Performance
- [ ] Core Web Vitals pass on mobile and desktop
- [ ] Images optimized (right format, compressed, sized, lazy-loaded)
- [ ] No render-blocking issues flagged
- [ ] Fonts load without layout shift

## 4. Functional QA
- [ ] Every link tested (nav, body, footer, CTAs)
- [ ] Every form tested with a real submission; confirmation + delivery verified
- [ ] Responsive on mobile / tablet / desktop breakpoints
- [ ] Cross-browser (Chrome, Safari, Firefox at minimum)
- [ ] Analytics installed and firing (GA/other)
- [ ] Search Console connected

## 5. Content QA
- [ ] No [CONFIRM], [DECISION NEEDED], [IMAGE], or placeholder text remaining
- [ ] No lorem ipsum
- [ ] Alt text on all meaningful images
- [ ] Spelling/grammar pass
- [ ] Client facts, names, numbers, testimonials verified (not AI-invented)

## Verdict
- **Ready to launch** / **Not ready** — list blocking items.
